# TASK-INDICATOR-DRAWDOWN-021：V8.9.0 指标回撤中心

## 模块定位

V8.9.0 接通主界面左侧既有“指标回撤”入口，打开独立、单实例、可缩放的纯本地只读指标回撤中心。该窗口用于审阅真实历史日线回撤，不建立第二套行情抓取、策略、账户或交易事实源。

## 首版标的范围

- 固定指数：`251.NDXTMC`、`100.NDX100`。
- 场内 ETF：当前 `strategy_config.enabled=true` 的策略代码，经证券代码规范化后去重。
- 同一 ETF 关联多个启用策略时只展示一行，关联策略代码稳定排序并以逗号分隔。
- 不包含 OTC、汇率、其它顶部指数、停用策略、仅持仓但未启用的代码、账户回撤或策略净值回撤。

## 指标口径

- 历史最高收盘、20/60/120/252 个有效交易点回撤、年初至今回撤和全历史标准最大回撤，全部只使用真实前复权日线 `Close`。
- 当前回撤公式为 `latestQuote / max(历史最高收盘, latestQuote) - 1`。最新真实 quote 只影响最新价和当前回撤，不写回历史、不进入历史最大回撤或周期回撤。
- quote 高于历史最高收盘显示 `0.00%（创新高）`；相等显示 `0.00%`；缺少有效 quote 时最新价和当前回撤显示 `--`，不得用历史末点代替。
- 标准最大回撤按时间顺序维护运行峰值；只有严格更高的 Close 才更新峰值，峰值相等保留首次日期，最大回撤相等保留首次谷底。少于两个有效点显示 `--`，单调上涨显示 `0.00%` 且区间显示 `--`。
- 20/60/120/252 日均指最后 N 个有效历史交易点；不足 N 点时使用现有全部真实点降级计算，并明确标记“数据不足”。
- 年初至今回撤使用历史最后一点所属年份，从该年首个有效点到最后一点计算；该年仅一个点时为 `0.00%`。

## 历史来源与质量

- ETF 历史首选 `TENCENT_DAILY_QFQ`，指数历史首选 `EASTMONEY_HISTORY`。
- 同一来源按 `updated_at`、`id` 选择较新候选；较新 payload 损坏时回退到同一来源较早的完整真实候选，再尝试另一已知真实日线来源。不同 payload 或来源不得拼接。
- 有效序列按日期排序，同日重复点保留 payload 中最后一个有效 Close；无效、非正数 Close 不参与计算。不得补点、插值或生成模拟历史。
- 完整历史签名包含 source、market type、symbol、row id、updated time、payload length 和 payload SHA-256。
- 历史截止日与参考日期相差超过 7 个自然日时标记“历史滞后”。参考日期优先使用有效 quote 日期，否则使用本地当前日期；该状态不触发网络请求。
- 数据状态优先级固定为：数据损坏、无历史、数据源异常、熔断冷却、限频、历史滞后、数据不足、实时行情缺失、正常。

## 只读刷新与性能边界

- 首次打开使用一个 SQLite `mode=ro` 连接、`PRAGMA query_only=ON` 和一个只读事务，批量读取启用策略、目标历史、目标 quote 及相关数据源状态。
- 首次完整历史解析、指标计算和主表绑定在 `Show()` 前完成，避免先显示空白窗口。
- 每 2 秒只重读当前标的、真实 quote 和相关状态，更新最新价、当前回撤及状态；不读取历史 payload，不重算全历史。
- 每 30 秒检查一次目标历史元数据签名。只有签名或启用标的集合变化时，才在后台读取并重算变化标的；窗口关闭会取消后续应用，失败时保留上一份不可变快照。
- 所有查询只覆盖目标标的，不读取 TradeLog、账户回放、持仓回放、策略决策、委托草案、runtime_log、分时缓存或无关历史。

## UI 与白闪边界

- 窗口尺寸 `1500 x 860`，最小尺寸 `1180 x 680`，使用原生标题栏、`ResizeMode=CanResize`、`ShowInTaskbar=False` 和 `#050B14` 深色背景。
- `SourceInitialized` 依次设置 DWM 深色标题栏/边框和 HWND CompositionTarget 深色背景。
- 不使用 `WindowStyle=None`、WindowChrome、透明窗口、Opacity 淡入、Reveal、启动遮罩、`Thread.Sleep` 或固定延迟。
- 深色横向/纵向滚动条仅在本窗口资源内定义；页面本身不横向滚动，18 列只读 DataGrid 自行滚动。
- 不提供手动刷新按钮，不复制 K 线窗口，不修改主界面旧回撤图。

## 摘要对账与选中详情

- 顶部固定保留 7 张摘要卡；第五张为“异常/缺失”，按最终互斥 `DataStatus` 汇总数据损坏、无历史、数据源异常、熔断冷却、限频和实时行情缺失。
- 摘要必须满足：监控标的 = 正常 + 数据不足 + 历史滞后 + 异常/缺失。同一行只按最终状态计数一次，不改变状态优先级，也不把限频标的伪装为正常。
- “异常/缺失”完整分类明细由快照构建器生成并通过 Tooltip 展示，XAML 不拼接状态业务口径。
- 底部选中标的详情为固定三行深色区域，依次展示身份信息、数据基础、回撤与状态；只绑定已有指标结果，长文本截断时通过 Tooltip 显示完整内容。
- 方法说明与选中详情保持分离；详情区不增加滚动条、图表、假数据、占位行或网络请求，主表继续固定 18 列并占用剩余可用空间。

## 安全边界

- 不新增网络请求，不绕过 `GlobalMarketRequestScheduler`，不修改 parser、router、scheduler、熔断或限频。
- 不写 SQLite，不写 runtime_log，不删除缓存，不修改数据库结构。
- 不读取或写入 TradeLog，不触发账户回放、策略计算、委托草案或成交。
- 不修改 `IndexDrawdownChartSeriesBuilder`、`SecurityChartWindow`、`KLineAggregator` 或主界面旧回撤图。
- V8.9.0 指标回撤中心采用标准 close-to-close 最大回撤；主界面旧回撤面板保持原锁定行为，本任务不修改。

## 人工验收

1. 左侧“指标回撤”可打开且重复点击保持单实例。
2. 普通打开、快速关闭重开、最大化/还原均无白闪。
3. 原生标题栏、深色边框及窗口专属深色滚动条正常。
4. 两个固定指数和启用 ETF 去重范围正确，18 列完整。
5. 类型筛选、搜索、排序、表格内部滚动及选中详情正常。
6. 当前回撤、创新高、周期回撤、YTD、最大回撤峰谷区间与真实数据一致。
7. 数据不足、历史滞后、无历史、损坏及缺 quote 状态清楚且不显示假 0。
8. 正常关闭后无残留进程，TradeLog、策略、委托、图表和主界面刷新不受影响。
