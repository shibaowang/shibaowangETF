## 范围

本任务只优化现有 `ManualDataEntryWindow` 中的 TradeLog 页面，不新增窗口、Tab、模块、业务字段、数据库表或业务操作。页面继续保留四个原操作：

1. 新增记录
2. 编辑选中
3. 删除选中
4. 保存全部

不增加搜索、筛选、重新加载、导入、导出、批量修改、复制交易或自动保存。

## 页面结构

TradeLog 页面使用专属标题、事实源说明、两行工具栏、字段图例和填满剩余空间的 DataGrid。工具栏左侧为新增、编辑、删除，右侧为保存；第二行显示当前记录数和纯 UI 编辑状态。

编辑状态仅包括：

- 已保存
- 有未保存修改
- 保存中
- 保存失败

这些状态不写 SQLite、不写 TradeLog、不写 `runtime_log`，也不会触发自动保存或关闭窗口确认。

## 视觉口径

- 所有新增样式均使用 `TradeLog` 专属 Key，不修改全局 Button 或 DataGrid 样式。
- 表头高度 38，行高 36，最小行高 34，单元格字号 13。
- DataGrid 保留内部横向和纵向滚动，不默认冻结列。
- 人工录入列使用深色编辑背景；系统计算列使用更暗的灰蓝色只读背景。
- 系统计算字段组依靠深色背景、灰蓝文字、只读属性和 ToolTip 区分，不使用净现金流前的橙色分隔线。
- 净现金流、本金、现金余额和总资产仅在显示层使用 N2 金额格式，不改变底层 `double` 值或数据库精度。
- 备注列使用 Star 宽度并保持约 180 的最小宽度，在最大化时吸收剩余空间；普通窗口继续使用 DataGrid 内部横向滚动。
- TradeLog 独立 Scope 隐藏公共内容标题、数据库路径和 TradeLog 标签头，页面内部仅保留“交易日志”主标题；原生标题栏仍为“交易日志录入”。
- 新增且未保存的 `Id == 0` 行使用低亮橙色提示，不提前分配数据库 ID。
- 数字转换错误继续使用 WPF Validation，并显示红色边框和原错误信息。
- 空集合时显示轻量空状态，不新增按钮、不访问行情、不写数据库。

## 严格保留的业务行为

以下业务链路不因本任务改变：

- `SaveTradeLogs`
- `SaveTradeLogsCore`
- `SafeCommitTradeLogGridEdits`
- `ValidateTradeLogForSave`
- `ReplayAccountFromTradeLogs`
- `CopyTradeLogCalculatedFields`
- `DeleteSelectedTradeLog`
- `TradeLogLedgerNormalizer`
- `AccountReplayService`
- `LocalDataRepository.SaveTradeLogsSnapshot`

保存仍由用户明确点击触发；保存前仍提交 DataGrid 编辑、自动计算金额、推演账务字段并逐条验证，保存后仍从 TradeLog 事实源执行账户回放。小数和 `.`, `0.`, `1.` 等合法中间输入状态继续依赖 LostFocus 转换；金额自动计算和人工覆盖能力均保留。

动作、来源和档位下拉项保持原值，不增加或删除任何选项。
