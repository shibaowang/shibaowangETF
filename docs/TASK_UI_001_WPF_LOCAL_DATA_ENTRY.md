# TASK-UI-001: WPF 本地数据录入与持久化

## 实现内容

- 在现有 WPF 主窗口基准布局内接入本地 SQLite 数据库。
- 新增本地数据记录模型：策略配置、账户状态、持仓、OTCMap、TradeLog、运行日志。
- 新增 `LocalDatabase` 初始化数据库、目录、表结构和轻量 schema version。
- 新增 `LocalDataRepository` 负责本地数据保存、读取、删除和运行日志写入。
- 新增深色风格手动录入窗口 `ManualDataEntryWindow`，通过系统设置入口、右上角齿轮和左侧真实模块入口打开。
- 左侧导航已接入真实入口：`溢价决策` 复用手动录入窗口并仅显示 `策略配置` / `OTCMap` / `底仓基准设置`，`交易日志` 复用手动录入窗口并仅显示 `TradeLog`；`系统设置` 和右上角齿轮仅显示只读 `系统维护` 说明页、`预警设置` 和 `界面快捷键设置`；底层保存、删除、校验、深色样式、列顺序持久化、底仓基准 app_settings 保存、预警 app_settings 保存和界面快捷键 app_settings 保存仍使用同一套窗口，不复制业务保存逻辑。
- 主窗口资产卡、ETF 表、TradeLog、账户状态、OTCMap 摘要改为读取本地数据库。
- 启动后自动进入本地运行状态，并启动随机间隔自动刷新框架。
- 无本地数据时显示 `--`、`未配置`、`暂无已配置 ETF`、`暂无交易日志` 等空态。

## 本地 SQLite 数据库路径

数据库文件固定创建在：

```text
%LocalAppData%\CrossETF.Terminal.UiShell.Reference\cross_etf_terminal.db
```

首次启动时程序会自动创建目录、数据库文件和表结构。

- `position_state`
- `otc_channel`

- `app_settings`
- `runtime_log`

## 手动录入入口

主窗口提供四个入口：

- 左侧导航的“溢价决策”：打开 `溢价决策配置`，显示“策略配置”“OTCMap”和“底仓基准设置”。
- 左侧导航的“交易日志”：打开 `交易日志录入`，显示“TradeLog”。
- 左侧导航的“系统设置”：打开 `系统设置 / 数据维护`，显示只读“系统维护”说明页、“预警设置”和“界面快捷键设置”。
- 右上角齿轮图标：与“系统设置”一致，打开 `系统设置 / 数据维护`。

系统设置和右上角齿轮不再显示任何业务数据手动录入页签，包括“账户状态”“持仓”“策略配置”“OTCMap”“底仓基准设置”“TradeLog”，只显示“系统维护”说明页、“预警设置”和“界面快捷键设置”。

## 预警设置

系统设置内容区域支持纵向滚动，窗口高度不足时也能看到完整预警设置、快捷键设置和说明文字。系统设置中提供 PushPlus 微信预警和本机系统语音提示的基础版配置：

```text
启用微信预警
PushPlus Token
保存预警设置
测试微信
启用系统语音
测试语音
重复提醒间隔
严重风险间隔
行情异常间隔
```

配置保存到既有 `app_settings`，不新增设置表：

```text
alert_pushplus_enabled
alert_pushplus_token
alert_voice_enabled
alert_repeat_interval_minutes
alert_severe_interval_minutes
alert_market_interval_minutes
```

预警事件由当前系统已经生成的策略决策、委托草案、行情源状态、账户回放状态和运行日志派生，不重新计算策略、不修改委托草案、不写入 TradeLog、不自动交易。PushPlus Token 为空时不发送测试微信并在窗口底部提示；发送失败或系统语音播放失败只记录状态和运行日志，不影响手动录入窗口其它保存功能。

## 界面快捷键

系统设置中提供简洁版隐藏 / 显示主界面的全局快捷键设置：

```text
显示/隐藏窗口    [ Alt+1 × ]
恢复默认设置
```

默认启用 `Alt+1`，内部保存为 `ui_hotkey_modifiers=Alt`、`ui_hotkey_key=D1`，界面显示为 `Alt+1`。用户点击快捷键胶囊后直接按新组合，注册成功即自动写入既有 `app_settings` 并立即生效，不再显示修饰键复选框、主键下拉框或保存按钮。

支持的修饰键为 `Ctrl`、`Alt`、`Shift`、`Win`，至少选择一个；主键支持 `A-Z`、`0-9` 和 `F1-F12`。配置保存到既有 `app_settings`：

```text
ui_hotkey_enabled
ui_hotkey_modifiers
ui_hotkey_key
```

程序使用 Windows 原生 `RegisterHotKey` / `UnregisterHotKey`，不使用全局键盘钩子。快捷键触发时，主窗口可见则 `Hide()`；主窗口隐藏则 `Show()`、恢复隐藏前窗口状态并 `Activate()` 到前台。隐藏不会停止行情刷新、账户回放或程序进程。注册失败或快捷键冲突时提示“快捷键冲突，未保存”，不覆盖旧配置。点击 `×` 会注销当前快捷键并保存为禁用；点击“恢复默认设置”会立即恢复并注册 `Alt+1`。

内部 `ManualEntryScope.All` 仍可用于测试或开发排查，正式 UI 入口不再暴露完整业务页签范围。

录入窗口支持本地新增、编辑、删除、保存。TradeLog 只保存和展示用户录入记录，不执行现金回放、交易计算或策略判断。

## 主界面数据化说明

主界面不再显示写死的资产、行情、ETF 行和 TradeLog 行。

- 顶部行情条统一显示 `-- / 未连接`。
- 顶部时间显示当前本机时间。
- 状态区显示本地数据库运行状态。
- 资产卡读取 `account_state` 最新记录。
- ETF 表读取 `strategy_config` 和 `position_state`。
- TradeLog 表读取 `trade_log`。
- OTCMap 摘要读取 `otc_channel`。
- 回撤监控区域只显示网格、坐标和空态说明，不绘制假行情曲线。

## 启动随机刷新框架

程序启动后自动启用 WPF `DispatcherTimer`。

- 每次刷新后重新随机下一次间隔。
- 间隔范围为 2 到 4 秒。
- 刷新内容只包括当前时间、本地数据库读取、UI 状态、最近刷新时间和本地刷新耗时。
- 数据库读取失败时主界面显示错误状态，并尝试写入 `runtime_log`。

## 为什么不提供手动刷新按钮

本模块要求启动后自动刷新本地状态，且禁止提供手动触发行情或数据刷新的入口。手动刷新按钮容易被误解为联网拉取行情或同步外部数据，所以本阶段不提供。

## 为什么不接入真实行情

真实行情联网、频率限制、接口容错和数据可信度属于后续模块 2 的范围。本模块只完成本地录入、保存、读取和 UI 数据化。

## 为什么不做模拟行情和模拟交易

模拟行情和模拟交易会让 UI 看起来像已经有真实运行结果，容易混淆本地录入数据和业务计算结果。本模块明确不使用假数据冒充运行结果。

## 人工验收步骤

1. 在项目根目录执行 `dotnet run --project .\CrossETF.Terminal.UiShell.Reference.csproj` 启动程序。
2. 确认主界面保持现有金融终端布局，左侧导航、顶部状态区、四张核心卡片、ETF 表格和底部模块没有破坏。
3. 点击左侧“溢价决策”，确认能打开“溢价决策配置”，且只显示“策略配置”“OTCMap”和“底仓基准设置”。
4. 关闭录入窗口后，点击左侧“交易日志”，确认能打开“交易日志录入”，且默认显示“TradeLog”。
5. 在“底仓基准设置”页签确认能看到“模式”“比例”“固定金额”和“保存底仓基准设置”。
6. 关闭录入窗口后，点击左侧“系统设置”，确认能打开“系统设置 / 数据维护”，且只显示“系统维护”只读说明页、“预警设置”和“界面快捷键设置”。
7. 关闭录入窗口后，点击右上角齿轮，确认行为与“系统设置”一致，不显示“账户状态”“持仓”“策略配置”“OTCMap”“底仓基准设置”“TradeLog”。
8. 在“策略配置”页签手动录入一条策略配置并点击保存。
9. 在“OTCMap”页签手动录入 OTCMap 通道并点击保存。
10. 在“交易日志”入口的“TradeLog”页签手动录入 TradeLog 记录并点击保存。
11. 关闭程序。
12. 重新执行 `dotnet run --project .\CrossETF.Terminal.UiShell.Reference.csproj` 启动程序。
13. 确认主界面能读取并显示刚才录入的策略配置、OTCMap 摘要和 TradeLog；账户状态和持仓由 TradeLog 回放派生，不再从系统设置手动维护。
14. 确认 UI 上没有“手动刷新”“刷新行情”“重新拉取”“同步行情”“拉取数据”“重新联网”等手动刷新按钮或入口。
15. 确认顶部指数行情仍显示 `-- / 未连接`，不显示假行情点位或假涨跌幅。
16. 确认图表区域只显示网格、坐标和空态说明，不显示假行情曲线。
