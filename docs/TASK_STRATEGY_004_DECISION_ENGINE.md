## 策略决策优先级

1. 前置异常：账户回放财务异常、策略停用、本金不可用、行情价格缺失时，不输出买卖建议。
2. 战略底仓：按 `base_position_mode` 计算底仓目标，当前账户持仓总成本低于目标减容差时优先输出 `战略底仓 / 逢低吸筹`，不再被极端溢价、普通溢价止盈、收益止盈、底仓保护、纯场外持股观察或禁建仓提前截断。
3. 极端溢价 / 禁建仓：底仓完成后再识别极端溢价；有场内 ETF 可卖超额成本时输出极端溢价卖出，无持仓时禁止建仓，可卖超额成本不足时保持底仓保护。
4. 普通溢价止盈 / 收益止盈：底仓完成后，且账户级可卖超额成本大于 0 时才输出减仓建议。
5. T1-T6 狙击：真实历史 K 前置就绪后，按指数回撤计算档位和目标金额。
6. 持仓观察：已有持仓但没有触发卖出、战略底仓、T1-T6 继续买入或禁建仓时，输出 `√ 持股待涨 / 正常趋势`；空仓且无操作时才输出 `等待建仓 / 空仓观察`。

## 战略底仓规则

- 本金基准来自账户回放 `principal`，没有 TradeLog 回放时使用手动账户状态。
- 底仓设置存放在既有 `app_settings`，不新增设置表：`base_position_mode=ratio|amount`，`base_position_ratio` 默认 `0.20`，`base_position_amount` 默认 `0`。
- `ratio` 模式下底仓目标为 `principal * base_position_ratio`；比例输入支持 `20%`、`20`、`0.20`，保存时统一为 `0.20`，并限制在 `0..1`。
- `amount` 模式下底仓目标为 `base_position_amount`；固定金额不能小于 0，超过本金时按本金封顶，策略状态不崩溃，真实狙击资金池保持非负，并在派生状态中标记 `base_target_capped`。
- 底仓容差为 `max(1, principal * 0.001)`。
- 当前底仓使用持仓剩余总成本金额，不使用市值。
- 目标金额为底仓缺口，并受当前可用现金上限约束。
- 底仓未完成时，主操作指令始终表达底仓不足和补底仓意图；如果后续委托层因现金、整手、OTCMap 日限额、净值缺失或通道不可用导致不可执行，委托金额仍按真实可执行量化结果展示，不用不可执行的理论缺口金额冒充委托。
- 本模块不计算最终股数，不写 TradeLog。

## 止盈和溢价规则

- `extra_price`：极端溢价阈值。
- `take_profit_price`：溢价止盈阈值。
- `sell_ratio`：收益止盈阈值。
- 溢价只属于场内 ETF 交易价格和实时估值之间的关系；场外 A/C 基金、场外替代持仓和场外替代买入没有“溢价交易”概念。
- 底仓未完成时，场内 ETF 溢价不能禁止场外替代补底仓，也不能触发场外替代持仓卖出。
- 底仓保护按账户级口径判断，不再用单只 ETF 成本对比账户底仓目标。
- 账户级可卖超额成本为 `max(0, account_total_position_cost - base_target_amount)`。
- 有持仓且触发极端溢价时，若账户级可卖超额成本大于 0，输出 `全清换现金(留底) / 极端溢价`；若可卖超额成本不足，输出 `-- / 底仓保护`。
- 无持仓且触发极端溢价时，输出 `极端溢价 / 禁止建仓`，该判断不依赖 T1-T6 档位是否触发。
- 溢价止盈和收益止盈的建议金额不超过当前标的成本和账户级可卖超额成本，最终股数 / 份额量化仍由模块 5 处理。
- 当普通溢价止盈或收益止盈命中，但账户底仓未完成导致账户级可卖超额成本为 0 时，不输出 `-- / 底仓保护`，而是继续进入战略底仓补齐、T1-T6 档位完成或持仓观察判断。
- 当策略只有场外替代持仓且没有场内 ETF 可优先释放时，普通溢价止盈不抢占 T1-T6 档位展示；若当前触发 T 档，按 VBA 口径显示 `一档建仓完成 / 场外替代` 等场外替代持有状态，不提前生成场外卖出或继续买入建议。
- 策略决策收益率与 ETF 表持仓收益率保持同一聚合口径：优先使用 `position_replay_state` 的策略级持仓市值；只有没有聚合持仓时才使用 `otc_position_replay_state` 明细，避免场外持仓重复计入。

- 指数回撤 `<= -10%`：狙击二档。
- 指数回撤 `<= -15%`：狙击三档。

## 权重和资金池

默认权重为：

- T1 = 1
- T2 = 2
- T3 = 4
- T4 = 8
- T5 = 16
- T6 = 32

总份数默认 `63`。某档配置权重小于等于 0 时，该档回退默认权重。

主界面真实狙击资金池规则：

- `base_target_amount` 为当前可配置底仓目标。
- `base_current_cost` 为当前剩余持仓总成本。
- `base_gap_amount = max(0, base_target_amount - base_current_cost)`。
- `real_sniper_pool = max(0, available_cash - base_gap_amount)`。
- 主界面“狙击资金池”卡片只显示 `real_sniper_pool` 和既有可用档位份数，不额外显示现金余额、预算池或调试字段。

T1-T6 内部档位预算基准规则：

- 如果最后一条 `tier=周期结束` 的 TradeLog 存在且该行 `cash_balance > 0`，使用该现金余额作为内部档位预算基准。
- 否则统计当前周期内 `action=买入` 且 `tier=战略底仓` 的实际买入金额。
- 如果实际战略底仓买入金额大于 `0.01`，使用该实际金额作为底仓占用；否则使用当前可配置底仓目标。
- 底仓占用按 `0..principal` 封顶，内部档位预算基准为 `principal - 底仓占用`，并限制为非负。
- 周期结束只认 `tier=周期结束`，不认 `action=周期结束` 或 `memo=周期结束`。
- 如果存在最后一条 `tier=周期结束`，本周期已执行狙击金额只统计该行之后的 `狙击一档` 到 `狙击六档` 买入。
- T1-T6 的累计目标按内部档位预算基准和权重份数计算，最终可建议金额仍受主界面真实狙击资金池上限约束。

## 今日/当日盈亏口径

今日盈亏 / 当日盈亏已锁定为北京时间自然日 `[今日 00:00:00, 明日 00:00:00)`。策略和展示不得直接使用行情源当日涨跌、涨跌幅、涨跌额或 `现价 - 昨收` 替代账户今日盈亏；该口径不改写 TradeLog，不自动生成成交，不改变委托草案边界。

## 自动刷新

策略决策在以下输入变化后自动重算：

- 程序启动读取本地库。
- TradeLog 保存并完成账户回放。
- 策略配置保存。
- 账户回放派生状态变化。
- 行情缓存或历史高点缓存变化。
- 主界面 2 到 4 秒刷新读取最新派生表。

后台计算防重入，同一时间只运行一个策略计算任务。计算失败写入 `runtime_log`，不导致程序闪退。
