## 实现

### 深度判断

`ChartHistoryDepthEvaluator` 使用真实本地聚合结果，输出日K、周K、月K数量及最早/最新日期：

- 月K不少于 120 根：理想深度；
- 月K不少于 60 根：基本充分，可计算月 MA60；
- 月K少于 60 根：月线深度不足；
- 周K少于 260 根：周线深度不足。

只有打开周K或月K且对应深度不足时才允许深补齐。日K普通刷新、滚轮、拖动、十字光标、MA开关和B/S更新不会触发深补齐。

### ETF 深历史

- 目标最多 3000 根，但每页继续请求腾讯 qfq 日K 320 根。
- 使用同一接口按结束日期向前串行分页，禁止并发，支持取消。
- 第一页走既有 `EtfDailyKLine` 调度；后续页继续通过 `GlobalMarketRequestScheduler.TryAcquireRaw` 执行 10～30 秒主机限频。
- 相邻页保留一个真实交易日重叠，并校验重叠日 OHLC 一致；没有重叠或复权值冲突时整次结果作废，绝不简单拼接不同复权批次。
- 整次分页任一页失败都不返回部分 payload，旧缓存继续使用。

### 指数深历史

- 继续使用东方财富 `beg=19900101&lmt=10000` 的既有 DailyHistory 路径。
- 不继续放大 `lmt`，不拼接其它指数或 ETF 数据。
- 真实源返回全部可用历史但仍少于目标时，显示“已显示数据源全部可用历史”。

### 去重与来源穷尽

- 同一协调器进程按来源和证券代码对成功、来源穷尽或临时失败结果去重；窗口取消会清理 `InProgress`，重新打开同一证券后允许再次尝试。
- 来源已穷尽时，把来源、数量、日期范围和检查时间写入现有 `app_settings`；30 天内且当前缓存仍覆盖该检查点时不重复请求。
- 缓存缺失、数量回退或日期范围回退时，旧检查点不会阻止重新检查。
- 不新增数据库表。

### 缓存替换保护

- 候选必须是 DailyLike，OHLC 有效，日期唯一且可严格排序。
- MonthlyLike、WeeklyLike、Sparse、Invalid 和显示点均不能替换真实日K。
- 明显落后、较短滚动窗口或未扩展覆盖范围的候选不覆盖更长缓存。
- ETF 候选只在腾讯来源中择优；指数候选只在东方财富来源中择优。
- 同一最新日期附近优先更早的起始日期和更多有效日K，而不是只按 `received_at` 选择。
- `QUOTE_INTRADAY_BAR` 仍只存在于显示快照，不写历史缓存。

## PATCH-CANCEL：窗口级取消

- `ChartWindowManager` 在创建每个 `SecurityChartWindow` 时创建独立 `ChartWindowLifetime`，其 CTS 链接到主程序 `_marketRefreshCts.Token`。
- `ChartSubscription.LifetimeToken` 随周期切换保留；协调器为外层刷新 Token 与窗口 Token 创建临时 linked token。
- 窗口关闭顺序为：窗口先完成关闭标记，随后取消 Lifetime、取消订阅、移除窗口注册、释放 Lifetime、解除事件引用。
- 取消一个窗口只终止该证券当前请求，协调器捕获取消后继续处理其它活动订阅；应用 Token 取消会终止全部窗口请求。
- 腾讯页面间 scheduler 等待、可取消 Delay、HTTP 请求、分页解析与重叠合并均使用窗口对应 Token。
- 最终质量校验、内存缓存写入、`market_history_cache` 保存、`app_settings` 检查点和 Snapshot 发布前均再次检查取消。
- 取消不记录 HTTP 失败，不增加 breaker/scheduler 失败计数，不写来源穷尽或成功检查点，不清空或覆盖原缓存，也不向已关闭窗口发布 Snapshot。
- 深补齐状态区分 `InProgress`、`Completed`、`SourceExhausted`、`TemporaryFailure` 与 `Cancelled`；`Cancelled` 不阻止同一进程重新打开后重试。
- 取消时只释放该证券的 scheduler 预约和协调器本地请求时间；主机与端点防封间隔及所有限频参数保持不变。
