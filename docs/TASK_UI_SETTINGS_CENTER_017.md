- 正式版本：`V8.5.0`
- 开发基线：`v8.4.0` / `3e4458e9c246fc4a8490f9d68a822366c120a823`

## 入口与窗口边界

- 主界面继续只保留现有一级入口“系统设置”。
- 入口继续打开 `ManualDataEntryWindow`，范围为 `ManualEntryScope.SystemSettings`，并保持原模态行为。
- 不新增独立设置窗口、Tab、一级导航项或业务按钮。
- `All` 范围的“系统维护”页复用同一套设置中心控件，不复制第二套内容。
- 保留原生标题栏、DWM 深色处理、窗口尺寸、Owner、CenterOwner 和 ShowInTaskbar 规则。

## 四个一级分类

系统设置中心固定为以下四项，默认打开“通用设置”：

1. 通用设置：快捷键、版本与程序目录。
2. 预警与通知：微信通知、系统语音与提醒频率。
3. 数据安全：状态摘要、备份记录、维护说明与恢复危险区。
4. 运行与诊断：系统诊断、运行健康、日志与报告。

四个页面在窗口构建阶段创建并缓存。菜单切换仅替换右侧页面引用、更新选中视觉并将该页滚动位置恢复到顶部，不重新创建页面、DataGrid 或事件处理程序，也不触发数据库、行情、备份或诊断刷新。

## 页面职责

### 通用设置

- 保留现有“显示/隐藏窗口”快捷键设置及恢复默认操作。
- 软件信息只展示产品名称、当前版本、FileVersion 和构建标识。
- 本地数据目录和系统边界说明保持只读；长路径使用省略显示和完整 ToolTip。

### 预警与通知

- 保留 PushPlus、系统语音、重复提醒、严重风险和行情异常间隔。
- 测试微信、测试语音和保存预警设置继续使用原 Click 处理程序。
- 不改变 Token、测试、语音或频率保存行为。

### 数据安全

固定顺序为：

1. 状态摘要；
2. 备份记录；
3. 数据库位置与维护边界；
4. 恢复数据库危险区。

备份和恢复继续复用原服务、完整性校验、双重确认、恢复标记和关闭流程。本任务不增加清库、重建、缓存清理或迁移功能。

### 运行与诊断

- 保留综合状态、行情概要、私有内存、界面延迟、系统诊断、运行健康及日志与报告。
- 继续读取现有诊断快照和运行健康数据，不联网、不写 SQLite、不复制诊断规则。
- 不为视觉效果改写真实状态。

## UI 规范

- 左侧导航宽度 `226`，导航项高度 `66`，图标 `19`，主标题 `15`，副标题 `12`，选中项保留 `3px` 指示条。
- 页面标题 `24px`，说明 `14px`，顶部边距 `28px`，统一内容最大宽度、卡片间距、边框和圆角。
- 页面只允许独立纵向滚动，不允许设置中心整体横向滚动。
- 主按钮用于“保存预警设置”和“立即备份”；“恢复选中备份”为危险按钮；其余现有按钮为次级按钮。
- 按钮样式仅在 `ManualDataEntryWindow` 局部生效。

## 业务锁定边界

本任务不修改数据库结构、备份恢复服务、预警发送服务、快捷键注册逻辑、诊断判断规则、运行健康服务、TradeLog、账户回放、策略、委托、行情、图表或主界面业务逻辑；不引入 `WindowChrome`、透明窗口、`Opacity=0`、启动遮罩、页面动画或 `Thread.Sleep`。
